0Bức tranh tổng thể — hệ thống gồm những mảnh nào?

Một website hiện đại không nằm gọn trong 1 chỗ. Project này dùng 4 dịch vụ, mỗi cái một vai trò:

Dịch vụVai tròMiễn phí?
GitHubKho lưu trữ code (như Google Drive nhưng cho code, có lịch sử thay đổi)Có
Supabase"Bộ não" phía sau: database lưu tài khoản người dùng, xử lý đăng nhập (OTP + Google)Có (gói Free)
VercelNơi website thực sự "sống" — biến code thành trang web ai cũng truy cập đượcCó (gói Hobby)
BrevoDịch vụ gửi email thật (để gửi mã OTP 6 số)Có (300 email/ngày)
Google Cloud ConsoleNơi đăng ký "Đăng nhập bằng Google" cho websiteCó

Luồng hoạt động: Code nằm trên GitHub → Vercel lấy code đó dựng thành website → Website gọi tới Supabase mỗi khi cần đăng nhập/lưu dữ liệu → Supabase gọi Brevo để gửi email OTP, và gọi Google để xác thực khi người dùng bấm "Đăng nhập với Google".

1Chuẩn bị máy tính

  1. Cài Node.js (bản LTS): nodejs.org — cứ Next, Next, Install, không cần tick gì đặc biệt.
  2. Cài Git: git-scm.com/downloads — cứ Next mặc định.
  3. Kiểm tra cài xong chưa: mở PowerShell, gõ: 
    node -v
git --version
    Thấy hiện số phiên bản (không phải lỗi "not recognized") là được.

2Tài khoản cần tạo (theo đúng thứ tự)

Nguyên tắc quan trọng: dùng 1 email của chính bạn xuyên suốt cho tất cả (GitHub, Supabase, Vercel, Brevo, Google) để dễ quản lý và không bị lẫn với tài khoản người khác.

2.1 Tạo tài khoản GitHub

  1. Vào github.com/signup
  2. Nhập email, đặt mật khẩu, đặt username (tên hiển thị, vd nguyenvana123) — nhớ kỹ vì dùng lại nhiều lần.
  3. Xác nhận email nếu được yêu cầu.

2.2 Tạo tài khoản Supabase

  1. Vào supabase.com → Start your project.
  2. Chọn Continue with GitHub → bấm Authorize.
  3. Nếu lần đầu, Supabase sẽ hỏi tạo Organization — đặt tên tuỳ ý, chọn plan Free.

2.3 Tạo tài khoản Vercel

  1. Vào vercel.com/signup
  2. Chọn Continue with GitHub → Authorize.

2.4 Tạo tài khoản Brevo (gửi email OTP thật)

  1. Vào brevo.com → Sign up free.
  2. Đăng ký bằng email, đặt mật khẩu, xác nhận email.
  3. Nếu được hỏi mục đích dùng, chọn đại loại "Transactional emails".

2.5 Tài khoản Google (cho nút "Đăng nhập với Google")

Chỉ cần 1 tài khoản Gmail bình thường — sẽ tạo "OAuth Client" ở mục 5.

3Lấy code project về máy

Nếu code đã có sẵn trên GitHub, clone về:

git clone https://github.com/<username>/logistics-dashboard.git
cd logistics-dashboard
npm install

Nếu làm từ đầu hoàn toàn (chưa có code), cần nhờ AI/lập trình viên viết code trước — phần này nằm ngoài phạm vi 1 file hướng dẫn setup.

4Dựng Supabase (database + đăng nhập)

4.1 Tạo project

  1. Vào supabase.com/dashboard → New project.
  2. Chọn Organization vừa tạo → đặt tên project (vd logistics-dashboard).
  3. Đặt 1 Database Password mạnh — lưu lại chỗ an toàn.
  4. Region chọn Southeast Asia (Singapore) — gần Việt Nam nhất.
  5. Bấm Create new project, đợi khoảng 1-2 phút.

4.2 Tạo bảng lưu thông tin người dùng

  1. Menu trái → SQL Editor → New query.
  2. Mở file supabase/schema.sql trong code project → copy toàn bộ → dán vào ô SQL Editor.
  3. Bấm Run (hoặc Ctrl+Enter). Thấy chữ "Success" là xong.
  4. Kiểm tra: vào Table Editor, sẽ thấy bảng tên profiles.

4.3 Lấy URL và anon key

  1. Project Settings (icon bánh răng) → API.
  2. Copy 2 giá trị: Project URL (dạng https://xxxxx.supabase.co) và anon public key (chuỗi dài bắt đầu eyJ...).
  3. Trong code project, mở/tạo file .env.local, điền: 
    NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
⚠️ Không chia sẻ service_role key (khác với anon key) cho bất kỳ ai — key đó có toàn quyền trên database. Chỉ dùng anon key cho website.

5Bật đăng nhập Google thật

5.1 Tạo OAuth Client trên Google Cloud

  1. Vào console.cloud.google.com
  2. Góc trên, chọn project → New Project → đặt tên tuỳ ý → Create.
  3. Tìm và mở "OAuth consent screen".
  4. Chọn External → điền App name, User support email → Save and Continue tới cuối.
  5. Credentials → Create Credentials → OAuth client ID.
  6. Application type: chọn Web application.
  7. Authorized JavaScript origins → thêm URL website (vd https://ten-app.vercel.app, hoặc tạm http://localhost:3000).
  8. Authorized redirect URIs → thêm: 
    https://<project-ref-supabase>.supabase.co/auth/v1/callback
    (<project-ref-supabase> là phần trong Project URL ở bước 4.3.)
  9. Bấm Create. Copy lại Client ID và Client secret hiện ra (secret chỉ hiện 1 lần).

5.2 Dán vào Supabase

  1. Authentication → Providers → tìm Google → bật Enable Sign in with Google.
  2. Dán Client ID và Client Secret → Save.

6Bật OTP 6 số qua email thật (dùng Brevo)

Mặc định Supabase chỉ cho gửi khoảng 2-3 email/giờ, và không cho sửa nội dung email trừ khi tự kết nối dịch vụ gửi email riêng (SMTP). Vì vậy cần Brevo.

6.1 Lấy thông tin SMTP từ Brevo

  1. Đăng nhập app.brevo.com
  2. Vào SMTP & API (Settings → SMTP & API).
  3. Tab SMTP: ghi lại SMTP Server (smtp-relay.brevo.com), Port (587), Login (dạng xxxxxxxxx@smtp-brevo.com).
  4. Mục SMTP Keys: bấm Generate a new SMTP key → copy giá trị (dạng xsmtpsib-...) — chỉ hiện 1 lần.

6.2 Cấu hình SMTP trong Supabase

  1. Project Settings → Authentication → mục SMTP Settings.
  2. Bật Enable Custom SMTP.
  3. Điền: Sender email (email đăng ký Brevo), Sender name, Host smtp-relay.brevo.com, Port 587, Username & Password lấy ở bước 6.1.
  4. Save.

6.3 Sửa email template để hiện mã 6 số (bắt buộc)

Vào Authentication → Emails, sửa 2 template (vì Supabase dùng template khác nhau cho "đăng ký lần đầu" và "đăng nhập lần sau"):

Template "Confirm signup"

<h2>Mã đăng nhập của bạn</h2>
<p>Nhập mã 6 số sau để đăng nhập:</p>
<p style="font-size:28px;font-weight:bold;letter-spacing:4px">{{ .Token }}</p>
<p>Mã có hiệu lực trong ít phút.</p>

Template "Magic Link"

<h2>Mã đăng nhập của bạn</h2>
<p>Nhập mã 6 số sau để đăng nhập:</p>
<p style="font-size:28px;font-weight:bold;letter-spacing:4px">{{ .Token }}</p>
<p>Mã có hiệu lực trong ít phút.</p>

→ Save cả 2.

6.4 Chỉnh độ dài mã OTP về 6 số

Vào Authentication → Settings, tìm mục Email OTP Length → đặt thành 6 (mặc định có thể là 8, sẽ lệch với giao diện web chỉ có 6 ô nhập).

7Khai báo URL cho Supabase biết web ở đâu

Authentication → URL Configuration:

  • Site URL: URL Vercel của bạn (vd https://ten-app.vercel.app)
  • Redirect URLs → thêm: 
    https://ten-app.vercel.app/**
http://localhost:3000/**
Nếu bỏ qua bước này, đăng nhập Google sẽ bị lỗi redirect, và verify OTP có thể không hoạt động đúng.

8Đưa code lên GitHub

Mở PowerShell trong thư mục project:

git init
git add .
git commit -m "Initial commit"
  1. Vào github.com/new
  2. Đặt tên repo (vd logistics-dashboard), chọn Public/Private tuỳ ý, không tick "Add README" → Create repository.
  3. Copy đoạn lệnh GitHub hiện ra, dạng: 
    git remote add origin https://github.com/<username>/logistics-dashboard.git
git branch -M main
git push -u origin main
    Dán vào PowerShell, Enter. Nếu được hỏi đăng nhập, đăng nhập qua trình duyệt hiện ra.
File .env.local (chứa key Supabase) đã được cấu hình để không bị đẩy lên GitHub (xem file .gitignore) — đây là chủ đích, giữ key riêng tư.

9Deploy lên Vercel

  1. Vào vercel.com/new
  2. Chọn Import cạnh repo logistics-dashboard.
  3. Vercel tự nhận diện Next.js, không cần đổi Build Settings.
  4. Mở mục Environment Variables, thêm đúng 2 biến (copy từ .env.local):
    KeyValue
    NEXT_PUBLIC_SUPABASE_URL(URL Supabase của bạn)
    NEXT_PUBLIC_SUPABASE_ANON_KEY(anon key của bạn)
  5. Bấm Deploy, đợi 1-2 phút. Xong sẽ có link dạng https://logistics-dashboard-xxxx.vercel.app.
Quay lại bước 5.1 và bước 7: cập nhật URL Vercel thật vào Google Console và Supabase — vì lúc đầu có thể bạn chưa có URL thật.

10Kiểm tra hoạt động

  1. Mở link Vercel → thấy trang Đăng nhập / Đăng ký.
  2. Thử tab Đăng ký: điền Họ tên, Email, Vai trò → bấm Tạo tài khoản.
  3. Kiểm tra hộp thư email (cả mục Spam) → có email chứa mã 6 số.
  4. Nhập mã vào 6 ô trên web → bấm Xác nhận → vào Dashboard.
  5. Thử nút Đăng nhập với Google → chọn tài khoản → tự động vào Dashboard.
  6. Kiểm tra dữ liệu: Supabase → Table Editor → bảng profiles → thấy dòng mới.

12Trạng thái hiện tại của project này

Đã setup thật, tham khảo:

GitHub: repo đã tạo, code đã push
Supabase: project + bảng profiles đã chạy
Brevo SMTP: đã kết nối, OTP 6 số
Google OAuth: đã bật
Vercel: đã deploy, env vars đã set
Nếu cần lấy lại thông tin (Project URL, Client ID...) để cấu hình máy mới, vào lại đúng dashboard tương ứng (Supabase Project Settings, Google Cloud Credentials, Vercel Project Settings) — không lưu lại các key/secret dạng chữ ở bất kỳ đâu ngoài các dashboard chính chủ đó.
